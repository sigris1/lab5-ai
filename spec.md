# URL Shortener Spec

## Что строим
- REST API на Flask для сокращения ссылок
- Endpoint `POST /shorten`: принимает JSON `{"url": "https://..."}`, возвращает `{"short_url": "http://localhost:5000/<code>"}`
- Endpoint `GET /<code>`: редирект (302) на оригинальную длинную ссылку
- Хранение данных в локальном SQLite (`urls.db`)
- Генерация короткого кода: 6 символов (base62)

## Что НЕ строим
- Веб-интерфейс (только API)
- Аутентификация / пользователи
- Срок жизни ссылок / удаление
- Аналитика кликов
- Валидация формата URL на уровне regex (только проверка на пустоту и строку)

## Acceptance Criteria
1. При валидном POST запросе создается запись в БД и возвращается короткий URL
2. При переходе по короткому URL происходит редирект на оригинал (HTTP 302)
3. Повторная отправка того же длинного URL возвращает уже существующий короткий код
4. Все endpoint'ы возвращают корректные HTTP статусы (200/201/302/400/404)

## Edge Cases
- Пустой или нестроковый `url` в теле запроса → `400 Bad Request`
- Переход по несуществующему коду → `404 Not Found`
- Ссылка уже была сокращена ранее → возврат существующего кода (идемпотентность)
- Длинный URL содержит спецсимволы и кириллицу → корректное сохранение и редирект
